Compliance Operations Playbook

Four fictional checklists for repeatable operations

Vendor Onboarding Checklist

  • ✓ Confirm business owner and vendor purpose
  • ✓ Collect security questionnaire
  • ✓ Review data access and risk level
  • ✓ Record approval decision

Policy Review Checklist

  • ✓ Confirm policy owner
  • ✓ Review last updated date
  • ✓ Capture reviewer comments
  • ✓ Publish approved version

Access Request Checklist

  • ✓ Validate requester and manager approval
  • ✓ Confirm least-privilege access
  • ✓ Set access review date
  • ✓ Document completion evidence

Audit Evidence Checklist

  • ✓ Identify control owner
  • ✓ Collect sample evidence
  • ✓ Check evidence date range
  • ✓ Mark ready for auditor review

The goal is to practice semantic HTML, responsive layout, Tailwind CSS, accessibility basics, and portfolio documentation.

Why use dummy compliance data?
